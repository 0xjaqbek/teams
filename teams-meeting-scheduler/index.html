🌟 Kluczowe funkcje

Automatyczne planowanie

Zaplanuj spotkania przez przyjazny interfejs webowy. System automatycznie dołączy do spotkań o wyznaczonej porze.

Obsługa polskiego Teams

Pełna kompatybilność z polską wersją Microsoft Teams. Automatyczne rozpoznawanie przycisków i interfejsu.

Transkrypcja w czasie rzeczywistym

Automatyczna transkrypcja polskich rozmów używając Google Speech API. Bez zapisywania audio - tylko tekst.

🚀 Instalacja krok po kroku

1

Zainstaluj wymagane oprogramowanie

A) Node.js

Node.js LTS

Runtime JavaScript dla automatyzacji

Pobierz Node.js
  1. Kliknij przycisk "Pobierz Node.js" powyżej
  2. Pobierz wersję LTS (Long Term Support)
  3. Uruchom pobrany plik i postępuj zgodnie z instrukcjami
  4. Po instalacji otwórz Command Prompt i wpisz: node --version
  5. Powinieneś zobaczyć numer wersji (np. v18.17.0)

B) Google Chrome

Google Chrome

Przeglądarka do automatyzacji Teams

Pobierz Chrome
  1. Pobierz i zainstaluj najnowszą wersję Chrome
  2. Ważne: Zaloguj się do swojego konta Microsoft Teams w Chrome
  3. Sprawdź czy możesz otworzyć Teams w przeglądarce
2

Pobierz projekt z GitHub

Opcja A: Pobierz ZIP (zalecana dla początkujących)

Teams Automation ZIP

Gotowy do użycia pakiet

Pobierz ZIP
  1. Kliknij przycisk "Pobierz ZIP" powyżej
  2. Zapisz plik na pulpicie lub w folderze Dokumenty
  3. Kliknij prawym przyciskiem na plik ZIP i wybierz "Wyodrębnij wszystko"
  4. Wybierz lokalizację (np. C:\teams-automation\)

git clone https://github.com/0xjaqbek/teams.git

Szybka instalacja zależności

  1. Otwórz folder z projektem
  2. Znajdź plik setup.bat
  3. Kliknij dwukrotnie na setup.bat
  4. Poczekaj aż instalacja się zakończy

Skrypt automatycznie zainstaluje wszystkie wymagane zależności

🚀 Uruchomienie - Wersja Lokalna

Gratulacje!

Wersja lokalna nie wymaga żadnej dodatkowej konfiguracji. Wszystko działa od razu po instalacji!

Szybki start - 4 proste kroki

1

Uruchom serwer lokalny

node local-joiner.js dashboard

Szczegółowe instrukcje dla osób początkujących:

Metoda 1: Win + R (Windows)
  1. Wciśnij jednocześnie Win + R (klawisz Windows + R)
  2. W okienku "Uruchom" wpisz: cmd
  3. Wciśnij Enter
  4. W czarnym oknie terminala wpisz komendę nawigacji:
    cd "C:\ścieżka\do\projektu\meeting-joiner"
    Zamień "ścieżka\do\projektu" na rzeczywistą lokalizację folderu
  5. Przykład: cd "C:\Users\TwojeImie\Desktop\teams-meeting-scheduler-main\meeting-joiner"
  6. Wciśnij Enter
  7. Teraz wpisz komendę: node local-joiner.js dashboard
  8. Wciśnij Enter
Metoda 2: Bezpośrednio z folderu (najłatwiejsza)
  1. Otwórz Eksplorator plików (folder żółty na pasku zadań)
  2. Przejdź do folderu meeting-joiner w pobranym projekcie
  3. Kliknij w pusty obszar paska adresu na górze okna (tam gdzie jest ścieżka)
  4. Usuń całą ścieżkę i wpisz: cmd
  5. Wciśnij Enter
  6. Terminal otworzy się automatycznie w odpowiednim folderze!
  7. Wpisz komendę: node local-joiner.js dashboard
  8. Wciśnij Enter

Sukces! Jeśli zobaczysz komunikat "Server running on http://localhost:3000" oznacza to, że wszystko działa poprawnie!

2

Otwórz dashboard w przeglądarce

http://localhost:3000

Serwer automatycznie utworzy lokalny dashboard pod powyższym adresem

3

Wprowadź swoje dane Teams

W dashboard przejdź do zakładki "Settings" i wprowadź:

  • Swoje imię i email
  • Email do konta Microsoft Teams
  • Hasło do konta Teams

Wszystkie dane są szyfrowane i przechowywane tylko lokalnie na Twoim komputerze

4

Zaplanuj pierwsze spotkanie

W zakładce "Dashboard":

  • Wklej link do spotkania Teams
  • Ustaw datę i godzinę
  • Kliknij "Schedule Meeting"

System automatycznie dołączy do spotkania w wyznaczonym czasie!

Uruchomienie przy starcie systemu

Dodaj skrypt do autostartu Windows, aby automatycznie uruchamiał się z systemem

1. Firebase - Baza danych i uwierzytelnianie

A) Utwórz projekt Firebase

  1. Przejdź do Firebase Console
  2. Kliknij "Utwórz projekt" lub "Create a project"
  3. Podaj nazwę projektu (np. "teams-automation-twoje-imie")
  4. Wyłącz Google Analytics (nie jest potrzebny)
  5. Kliknij "Utwórz projekt"

B) Skonfiguruj Firestore Database

  1. W konsoli Firebase kliknij "Firestore Database"
  2. Kliknij "Utwórz bazę danych"
  3. Wybierz "Rozpocznij w trybie testowym"
  4. Wybierz lokalizację (np. europe-west)

C) Włącz uwierzytelnianie

  1. Kliknij "Authentication" w menu
  2. Wybierz kartę "Sign-in method"
  3. Włącz "Email/Password"
  4. Włącz "Google" (opcjonalne)

D) Uzyskaj klucze konfiguracyjne

  1. Kliknij ikona koła zębatego → "Project settings"
  2. Przewiń w dół do sekcji "Your apps"
  3. Kliknij ikonę web () aby dodać aplikację web
  4. Podaj nazwę aplikacji i kliknij "Register app"
  5. Skopiuj konfigurację Firebase (będzie potrzebna później)

2. Google Cloud - Speech-to-Text API (opcjonalne)

To jest opcjonalne! Potrzebne tylko jeśli chcesz transkrypcję na żywo.

A) Utwórz projekt Google Cloud

  1. Przejdź do Google Cloud Console
  2. Kliknij "Wybierz projekt" → "Nowy projekt"
  3. Podaj nazwę projektu
  4. Kliknij "Utwórz"

B) Włącz Speech-to-Text API

  1. W Google Cloud Console przejdź do "APIs & Services" → "Library"
  2. Wyszukaj "Speech-to-Text API"
  3. Kliknij na API i wybierz "Enable"

C) Utwórz klucz API

  1. Przejdź do "APIs & Services" → "Credentials"
  2. Kliknij "Create Credentials" → "API Key"
  3. Skopiuj wygenerowany klucz API
  4. Opcjonalnie: Kliknij "Restrict Key" i ogranicz do Speech-to-Text API

3. Konfiguracja plików

A) Skonfiguruj web dashboard

  1. Otwórz plik web-dashboard/script.js
  2. Znajdź sekcję z firebaseConfig
  3. Zastąp przykładowymi danymi swoją konfigurację Firebase
const firebaseConfig = { apiKey: "twój-api-key", authDomain: "twój-projekt.firebaseapp.com", projectId: "twój-projekt-id", // ... reszta konfiguracji };

B) Skonfiguruj local-joiner

  1. Otwórz plik meeting-joiner/local-joiner.js
  2. Znajdź sekcję CONFIG na górze pliku
  3. Uzupełnij swoimi danymi:
const CONFIG = { FIREBASE_PROJECT_ID: 'twój-projekt-id', FIREBASE_PRIVATE_KEY: 'twój-private-key', FIREBASE_CLIENT_EMAIL: 'twój-client-email', ENCRYPTION_KEY: 'twój-bezpieczny-klucz', GOOGLE_API_KEY: 'twój-google-api-key' // opcjonalne };

🔧 Rozwiązywanie problemów

Chrome nie otwiera się z moim kontem

Rozwiązanie:

  1. Zamknij wszystkie okna Chrome
  2. W pliku local-joiner.js ustaw useRealProfile = true
  3. Sprawdź czy profileName odpowiada Twojemu profilowi Chrome
  4. Uruchom ponownie automatyzację

Błąd "node nie jest rozpoznany"

Rozwiązanie:

  1. Sprawdź czy Node.js jest zainstalowany: otwórz nowy Command Prompt
  2. Wpisz node --version
  3. Jeśli nadal błąd, przeinstaluj Node.js z nodejs.org
  4. Po instalacji uruchom nowy Command Prompt

Nie może dołączyć do spotkania

Sprawdź:

  1. Link do spotkania Teams jest poprawny
  2. Czas spotkania jest ustawiony w przyszłości
  3. Dane logowania Teams są poprawne
  4. Chrome ma dostęp do mikrofonu/kamery

Transkrypcja nie działa

Sprawdź:

  1. Google API Key jest poprawnie skonfigurowany
  2. Speech-to-Text API jest włączone w Google Cloud
  3. Mikrofon ma pozwolenia w Chrome
  4. W pliku CONFIG jest ustawiony GOOGLE_API_KEY

Firebase błędy uprawnień

Rozwiązanie:

  1. Sprawdź czy Firestore Database jest w trybie testowym
  2. Upewnij się że uwierzytelnianie Email/Password jest włączone
  3. Zweryfikuj konfigurację Firebase w script.js
  4. Sprawdź czy klucze Firebase są poprawnie skopiowane

Skrypt nie widzi zaplanowanych spotkań

Sprawdź:

  1. Plik user-credentials.json jest w folderze meeting-joiner
  2. Konfiguracja Firebase w local-joiner.js jest poprawna
  3. Spotkania są zaplanowane na przyszły czas
  4. Status spotkań to "pending" w dashboardzie

Problemy specyficzne dla wersji lokalnej

Nie mogę otworzyć terminala

Rozwiązanie krok po kroku:

  1. Sprawdź system: Naciśnij Win + R, wpisz "winver" i sprawdź wersję Windows
  2. Alternatywna metoda: Kliknij prawym przyciskiem na przycisk Start → "Windows PowerShell"
  3. W PowerShell: Użyj tych samych komend co w CMD (działają identycznie)
  4. Jeszcze inna opcja: Pobierz Windows Terminal z Microsoft Store (nowocześniejsze)

Dashboard localhost:3000 nie działa

Sprawdź kolejno:

  1. Czy terminal z komendą node local-joiner.js dashboard nadal działa
  2. Czy widzisz komunikat "Server running on http://localhost:3000"
  3. Spróbuj odświeżyć stronę (Ctrl + F5)
  4. Sprawdź czy antywirus nie blokuje połączenia lokalnego
  5. Spróbuj inny port: node local-joiner.js dashboard --port 3001

Dane nie zapisują się lokalnie

Problemy z uprawnieniami do zapisu:

  1. Sprawdź czy folder meeting-joiner/data istnieje (jeśli nie, zostanie utworzony automatycznie)
  2. Uruchom terminal jako administrator (kliknij prawym na CMD → "Uruchom jako administrator")
  3. Sprawdź uprawnienia folderu - kliknij prawym na folder → Właściwości → Zabezpieczenia
  4. Alternatywnie: Przenieś cały projekt do folderu Dokumenty

Automatyzacja nie startuje przy uruchomieniu

Konfiguracja autostartu Windows:

  1. Naciśnij Win + R, wpisz shell:startup
  2. Utwórz plik .bat z zawartością:
    cd "C:\ścieżka\do\meeting-joiner"
    node local-joiner.js schedule
  3. Zapisz jako teams-automation.bat w folderze Startup
  4. System automatycznie uruchomi skrypt przy starcie Windows
